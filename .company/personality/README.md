# 开发流程人格组合指南

本文档说明如何在软件开发的不同阶段使用对应的人格模板。

## 人格与阶段映射

| 阶段 | 人格 | 核心能力 | 输出物 |
|------|------|----------|--------|
| **需求创造** | 创新型 | 用户洞察、问题发现、价值定义 | 需求文档、用户故事、价值主张 |
| **设计** | 系统型 | 架构设计、模块划分、技术选型 | 架构图、接口设计、技术方案 |
| **开发** | 工匠型 | 代码实现、质量保证、技术攻坚 | 高质量代码、单元测试、技术文档 |
| **评审** | 严谨型 | 质量把关、风险识别、标准检查 | 审查报告、问题清单、改进建议 |

## 完整开发流程示例

### 阶段1：需求创造（创新型）
**角色**: 创新型产品经理
**任务**:
- 用户调研和需求挖掘
- 问题定义和价值分析
- 功能优先级排序
- 创建用户故事和验收标准

**输出**:
- 需求文档（PRD）
- 用户故事地图
- 原型设计

### 阶段2：设计（系统型）
**角色**: 系统型架构师
**任务**:
- 系统架构设计
- 模块划分和接口定义
- 技术选型和方案评估
- 非功能性需求设计

**输出**:
- 架构设计文档
- 接口规范
- 数据库设计
- 技术方案

### 阶段3：开发（工匠型）
**角色**: 工匠型开发者
**任务**:
- 编写高质量代码
- 实现单元测试
- 代码重构优化
- 技术文档编写

**输出**:
- 功能代码
- 单元测试
- 集成测试
- 技术文档

### 阶段4：评审（严谨型）
**角色**: 严谨型审查员
**任务**:
- 代码审查
- 安全审计
- 性能评估
- 质量检查

**输出**:
- 审查报告
- 问题清单
- 改进建议
- 质量评分

## 团队配置建议

### 小型团队（3-4人）
```
创新型产品经理 → 系统型架构师 → 工匠型开发者（兼评审）
```

### 中型团队（5-8人）
```
创新型产品经理
    ↓
系统型架构师
    ↓
工匠型开发者 × 2-3
    ↓
严谨型审查员
```

### 大型团队（9+人）
```
创新型产品经理 + 协调型项目经理
    ↓
系统型架构师 + 分析型技术专家
    ↓
工匠型开发者 × 4-6 + 务实型开发者 × 2
    ↓
严谨型审查员 + 分析型测试工程师
```

## 人格互补策略

### 创新型 + 严谨型
- **互补**: 创新探索 ↔ 风险控制
- **场景**: 新产品开发，需要平衡创新和稳定性
- **协作**: 创新型提出方案，严谨型评估风险

### 系统型 + 工匠型
- **互补**: 架构设计 ↔ 实现细节
- **场景**: 复杂系统开发，需要架构和实现的紧密配合
- **协作**: 系统型定义框架，工匠型精细实现

### 工匠型 + 严谨型
- **互补**: 质量追求 ↔ 质量把关
- **场景**: 高质量代码交付，双重质量保障
- **协作**: 工匠型自我要求，严谨型外部审查

## 使用建议

### 1. 根据项目阶段选择人格
- 早期探索：创新型为主
- 架构设计：系统型为主
- 功能开发：工匠型为主
- 质量保障：严谨型为主

### 2. 考虑人格互补
- 避免团队人格过于单一
- 确保创新和稳定的平衡
- 兼顾速度和质量

### 3. 动态调整
- 根据项目需求调整人格组合
- 关键阶段增加对应人格的比重
- 保持团队的灵活性

### 4. 培养多面性
- 鼓励员工发展多种人格特质
- 在不同阶段切换人格模式
- 提升团队的适应能力

## 创建员工示例

使用 `/hire-employee` 命令组合职位和人格：

```bash
# 创新型产品经理
/hire-employee position=product-manager personality=innovative-type name=张三

# 系统型架构师
/hire-employee position=architect personality=systematic-type name=李四

# 工匠型开发者
/hire-employee position=developer personality=craftsman-type name=王五

# 严谨型审查员
/hire-employee position=reviewer personality=rigorous-type name=赵六
```

## 注意事项

1. **人格不是限制**: 人格模板是指导而非限制，员工可以根据需要灵活调整
2. **情境适应**: 同一个人在不同情境下可以展现不同人格特质
3. **持续发展**: 鼓励员工发展多种人格能力，提升综合素质
4. **团队和谐**: 注意人格互补，避免冲突，促进协作
