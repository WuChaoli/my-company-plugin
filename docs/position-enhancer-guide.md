# 职位模板增强器 - 使用指南

## 概述

`position-enhancer` 是一个工具,用于增强现有职位模板的详细程度,使其字数从300-400字增加到600-800字,从而让AI更好地理解职位的职责、工作流程和协作方式。

## 快速开始

### 方法1: 使用命令(推荐)

```bash
/position-enhancer
```

然后按照提示选择要增强的职位模板。

### 方法2: 使用 skill

```bash
/skill position-enhancer
```

```markdown
## 工作流程
```

```markdown
2. **测试计划**: 根据需求制定测试策略,确定测试范围、测试类型(功能/性能/安全)、测试环境
3. **用例设计**: 设计测试用例覆盖正常场景、边界场景和异常场景,使用等价类划分和边界值分析方法
4. **测试准备**: 准备测试数据,搭建测试环境,配置测试工具
5. **测试执行**: 按照测试用例执行测试,记录测试结果和发现的问题
6. **缺陷报告**: 对发现的问题编写详细的bug报告,包含复现步骤、预期结果、实际结果
7. **回归测试**: bug修复后进行回归测试,确保修复有效且未引入新问题
8. **测试总结**: 编写测试报告,包含测试覆盖率、缺陷统计、质量评估

### 特殊场景
- **需求不明确**: 向需求规划师澄清,不在假设下继续测试
- **测试时间不足**: 优先测试核心功能和高风险场景,使用风险优先级排序
- **环境不稳定**: 记录环境问题,必要时暂停测试并协调解决
- **频繁回归**: 建立自动化测试套件,提高回归测试效率
```

## 使用场景

### 何时使用 position-enhancer

✅ **应该使用**:
- 发现职位模板描述不够详细,AI理解有偏差
- 需要让AI更准确地执行职位职责
- 职位模板字数少于500字
- 新员工激活后行为不符合预期

❌ **不应该使用**:
- 职位模板已经非常详细(>700字)
- 需要改变职位的基本结构(应该使用 position-creator 重新创建)
- 只是想更新职位模板的版本号

## 增强后的好处

1. **更好的AI理解**: 详细的描述让AI更准确地理解职位的方方面面
2. **更准确的执行**: 明确的流程和标准让AI执行更符合预期
3. **更好的协作**: 详细的协作接口说明让跨角色协作更顺畅
4. **更清晰的标准**: 明确的交付标准和验证命令让质量可控
5. **更少的误解**: 详细的场景说明和决策树减少模糊地带

## 批量增强所有职位模板

```bash
# 检查所有职位模板的字数

# 逐个增强

# ... 重复直到所有模板都增强完成
```

## 常见问题

### Q: 增强后会覆盖原文件吗?
A: 会的。建议在增强前先备份原文件,或者在git中提交当前版本。

### Q: 增强后可以手动调整吗?
A: 可以。增强后的模板可以继续手动编辑,position-enhancer 只是提供初始增强。

### Q: 增强会影响已创建的员工吗?
A: 不会。员工档案是通过引用方式使用职位模板的,职位模板的更新会自动生效。

### Q: 字数超过800字怎么办?
A: 可以手动精简内容,或者增强时选择性地只增强部分章节。

### Q: 增强后不满意怎么办?
A: 使用 `git checkout` 恢复到增强前的版本,然后手动调整或者再次增强。

## 技术细节

### 增强策略

position-enhancer 不会改变职位模板的章节结构,而是在每个章节中增加更详细的描述:

1. **概述**: 增加工作场景和角色区别
2. **职责**: 每条职责增加说明和交付标准
3. **工具**: 增加使用场景说明
4. **文档产出**: 增加内容模板和格式示例
5. **工作规则**: 增加理由和适用场景
6. **工作流程**: 增加详细步骤和决策树
7. **质量检查**: 增加验证标准和检查命令
8. **协作接口**: 增加详细流程和交接内容

### 字数分配

增强后的字数分配(总计600-800字):
- YAML frontmatter: ~50字
- 概述: 60-80字
- 职责: 150-180字
- 工具: 60-80字
- 技能: ~40字
- 文档产出: 180-220字
- 工作规则: 100-120字
- 工作流程: 200-250字
- 质量检查: 80-100字
- 协作接口: 120-150字
