# 员工档案内容规范

## 内容长度要求

员工档案总字数控制在 **2000字以内**，确保包含所有关键操作性指导。

## 完整性原则

### 核心理念
员工档案应包含职位模板中的所有关键信息，特别是操作性指导（可用Agents、文档产出、工作流程、质量检查、协作接口），这些信息对员工实际工作至关重要。

### 档案结构（12个部分）

1. **基本信息**（50字）
2. **职责**（200字）- 核心职责3-5条
3. **工具**（50字）- 分必需和可选
4. **技能**（30字）- 列表格式
5. **可用Agents**（100字）- 推荐agents列表
6. **文档产出**（500字）- 输入/输出表格+路径说明
7. **工作流程**（400字）- 标准流程+特殊场景
8. **人格特征**（100字）- 核心+风格
9. **工作规则**（150字）- 核心规则3-5条
10. **质量检查**（100字）- checkbox清单
11. **协作接口**（200字）- 按角色分组
12. **创建日期**（10字）

**总计**: 约1890字

## 内容提取指南

### 1. 基本信息
直接填充员工的基本信息：
```markdown
## 基本信息
- **ID**: [员工ID]
- **职位**: [职位中文名]
- **人格**: [人格中文名]
- **入职**: [入职日期]
```

### 2. 职责
从职位模板的"核心职责"部分提取3-5条：
```bash
# 提取核心职责
sed -n '/^### 核心职责/,/^### /p' .company/position/[职位].md | grep "^-" | head -5
```

### 3. 工具
从职位模板的"工具"部分提取，保持"必需"和"可选"分类：
```bash
# 提取工具部分
sed -n '/^## 工具/,/^## /p' .company/position/[职位].md | head -n -1
```

**格式示例**：
```markdown
## 工具
**必需**: Read, Write, Bash, Grep
**可选**: Glob, Edit, AskUserQuestion
```

### 4. 技能
从职位模板的"技能"部分提取：
```bash
# 提取技能
sed -n '/^## 技能/,/^## /p' .company/position/[职位].md | grep "^\\*\\*" | sed 's/.*: //' | tr '\n' ', '
```

### 5. 可用Agents（新增）

### 8. 人格特征

```bash

# 提取工作风格
sed -n '/^## 工作风格/,/^## /p' .company/personality/[人格].md | tail -n +2 | head -n -1
```

**格式示例**：

## 格式保持要求

### 1. Markdown表格
保持表格的完整格式，包括表头和分隔线：
```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 值1 | 值2 | 值3 |
```

### 2. 编号列表
保持编号和层级结构：
```markdown
1. **第一步**: 描述
2. **第二步**: 描述
   - 子步骤1
   - 子步骤2
```

### 3. Checkbox清单
保持checkbox格式：
```markdown
- [ ] 未完成项
- [x] 已完成项
```

### 4. 标题层级
保持标题的层级关系：
```markdown
## 二级标题
### 三级标题
#### 四级标题
```

## 注意事项

1. **完整性优先**: 不要为了简洁而省略关键信息
2. **格式一致**: 保持markdown格式的正确性
3. **结构清晰**: 使用标题和列表提高可读性
4. **表格完整**: 复制表格时保持所有列和行
5. **编号保持**: 流程步骤保持原有编号
6. **checkbox保留**: 质量检查清单保持checkbox格式
