# 人格模板组件说明

本文档说明人格模板各部分的用途和编写指导。

## 1. YAML Frontmatter

**用途**: 人格模板的元数据

**字段说明**:
- `personality`: 人格ID（英文小写+连字符，如：analytical-type）
- `name`: 人格名称（中文）
- `category`: 人格分类（思维型/行动型/社交型/平衡型）

## 2. 核心特征

**用途**: 用3-5个关键词快速识别人格特点

**编写指导**:
- 使用简洁的关键词
- 关注"如何做事"而非"做什么事"
- 保持中性客观的描述

**示例**:
- 分析型：逻辑严谨、数据驱动、系统思维、谨慎求证
- 创意型：发散思维、创新导向、灵活多变、勇于尝试

## 3. 工作风格

**用途**: 描述这个人格在工作中的典型表现

**包含维度**:
- **沟通**: 表达和倾听的方式
- **决策**: 做决策的依据和速度
- **协作**: 与他人协作的偏好
- **节奏**: 工作的节奏和计划性

**编写指导**:
- 使用对比词描述（如：直接/委婉）
- 简洁明了，便于快速识别
- 关注行为特征而非价值判断

## 4. 适配建议

**用途**: 指导如何将人格与职位组合，以及团队配置

**包含内容**:
- **适合职位**: 这个人格与哪些职位组合效果好
- **互补人格**: 与哪些人格形成互补
- **团队角色**: 在团队中通常扮演什么角色

**编写指导**:
- 提供具体的职位示例
- 说明互补的原因
- 描述在团队中的价值

## 编写原则

1. **简洁性**: 每个部分保持简洁，避免冗长描述
2. **中性性**: 不带价值判断，客观描述特征
3. **通用性**: 与具体职位无关，可广泛组合
4. **可识别性**: 特征描述具体，便于识别和应用
5. **实用性**: 提供可操作的组合建议
